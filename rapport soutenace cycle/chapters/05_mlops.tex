\section{Architecture des modèles et fondements mathématiques}
% ─────────────────────────────────────────────────────────────

\subsection{YOLO v11 — Détection d'objets}

\subsubsection{Principe général}

YOLO (\textit{You Only Look Once}) \cite{redmon2016yolo} réalise la détection d'objets
en \textbf{une seule passe forward} du réseau, sans étape de génération de propositions
de régions (contrairement à Faster R-CNN). Le réseau divise l'image d'entrée
$I \in \mathbb{R}^{W \times H \times 3}$ en une grille $S \times S$ de cellules.
Chaque cellule prédit $B$ boîtes englobantes et $C$ probabilités de classes.

\subsubsection{Formules de détection}

Pour chaque boîte $b$ dans la cellule $(i,j)$, le réseau prédit :

\begin{align}
\hat{x}, \hat{y} &= \sigma(t_x) + c_x,\quad \sigma(t_y) + c_y \label{eq:xy} \\
\hat{w}, \hat{h} &= p_w \cdot e^{t_w},\quad p_h \cdot e^{t_h} \label{eq:wh} \\
\hat{c}          &= \sigma(t_c) \label{eq:conf}
\end{align}

où $(t_x, t_y, t_w, t_h, t_c)$ sont les sorties brutes du réseau, $(c_x, c_y)$ les
coordonnées de la cellule, $(p_w, p_h)$ les dimensions de l'ancre de référence, et
$\sigma$ la fonction sigmoïde.

La \textbf{fonction de perte YOLO} est composée de trois termes :

\begin{equation}
\mathcal{L} = \underbrace{\lambda_{\text{coord}} \sum_{i,b} \mathbb{1}^{\text{obj}}_{ib}
  \left[(x_i - \hat{x}_i)^2 + (y_i - \hat{y}_i)^2\right]}_{\text{Localisation}}
+ \underbrace{\sum_{i,b} \mathbb{1}^{\text{obj}}_{ib}(c_i - \hat{c}_i)^2}_{\text{Confiance}}
+ \underbrace{\lambda_{\text{noobj}}\sum_{i,b}\mathbb{1}^{\text{noobj}}_{ib}(c_i - \hat{c}_i)^2
  + \sum_i \mathbb{1}^{\text{obj}}_i \sum_c (p_i(c) - \hat{p}_i(c))^2}_{\text{Classification}}
\label{eq:yolo_loss}
\end{equation}

\subsubsection{Intersection over Union (IoU)}

La métrique \textbf{IoU} mesure le chevauchement entre la boîte prédite $\hat{B}$
et la vérité terrain $B$ :

\begin{equation}
\boxed{
\text{IoU}(\hat{B}, B) = \frac{|\hat{B} \cap B|}{|\hat{B} \cup B|}
= \frac{\text{Aire de l'intersection}}{\text{Aire de l'union}}
\in [0, 1]
}
\label{eq:iou}
\end{equation}

\subsubsection{Métriques d'évaluation YOLO}

Soient $TP$ (vrais positifs), $FP$ (faux positifs), $FN$ (faux négatifs) :

\begin{align}
\text{Précision} &= \frac{TP}{TP + FP} \label{eq:precision} \\[4pt]
\text{Rappel}    &= \frac{TP}{TP + FN}  \label{eq:recall}   \\[4pt]
F_1              &= 2 \cdot \frac{\text{Précision} \times \text{Rappel}}
                              {\text{Précision} + \text{Rappel}} \label{eq:f1}
\end{align}

L'\textbf{Average Precision} (AP) par classe $c$ est l'aire sous la courbe
Précision-Rappel (PR) :

\begin{equation}
AP_c = \int_0^1 P_c(r)\,dr \;\approx\; \sum_{k} \bigl(R_{k+1} - R_k\bigr) \cdot P_{k+1}
\label{eq:ap}
\end{equation}

Le \textbf{mAP50} est la moyenne des $AP_c$ sur toutes les $|C|$ classes, avec un
seuil IoU de 0.50 :

\begin{equation}
\boxed{
\text{mAP50} = \frac{1}{|C|} \sum_{c \in C} AP_c \;\Big|_{\text{IoU} \geq 0.50}
}
\label{eq:map50}
\end{equation}

Pour \farmAI{} : $|C| = 40$ classes sur 8 modèles.

\subsection{Modèle IsolationForest — Détection d'anomalies IoT}

\subsubsection{Principe}

L'IsolationForest \cite{liu2008isolation} isole les anomalies par des coupes binaires
aléatoires dans l'espace des features. Une anomalie nécessite \textbf{moins de coupes}
pour être isolée qu'un point normal (elle est plus « rare » et « différente »).

\subsubsection{Score d'anomalie}

Le score d'anomalie pour un point $\mathbf{x}$ avec $n$ observations est :

\begin{equation}
\boxed{
s(\mathbf{x}, n) = 2^{-\dfrac{\mathbb{E}[h(\mathbf{x})]}{c(n)}}
}
\label{eq:isof_score}
\end{equation}

où :
\begin{itemize}
  \item $h(\mathbf{x})$ = profondeur d'isolation moyenne (nombre de coupures pour isoler $\mathbf{x}$)
  \item $c(n) = 2H(n-1) - \dfrac{2(n-1)}{n}$ = profondeur moyenne d'un arbre aléatoire
  \item $H(k) = \ln(k) + \gamma_E$ = nombre harmonique ($\gamma_E \approx 0.5772$)
\end{itemize}

\noindent\textbf{Interprétation :}
$s \to 1$ : anomalie certaine \quad $s \to 0.5$ : point normal \quad $s \to 0$ : très normal.

\subsubsection{Silhouette Score}

La qualité de séparation inliers/anomalies est mesurée par le Silhouette Score :

\begin{equation}
s(i) = \frac{b(i) - a(i)}{\max\bigl(a(i),\, b(i)\bigr)}, \qquad s(i) \in [-1, +1]
\label{eq:silhouette}
\end{equation}

\begin{equation}
\boxed{
\bar{S} = \frac{1}{n} \sum_{i=1}^n s(i) = 0.73 \quad \Rightarrow \quad \text{Structure forte}
}
\label{eq:sil_result}
\end{equation}

\subsection{RAG — Retrieval-Augmented Generation}

\subsubsection{Pipeline mathématique}

Le système RAG \cite{lewis2020rag} combine un encodeur sémantique et un LLM :

\begin{enumerate}
  \item \textbf{Encodage} : la question $q$ est encodée en vecteur
        $\mathbf{e}_q = \text{Encoder}(q) \in \mathbb{R}^{768}$
        (modèle \code{paraphrase-multilingual-mpnet}).

  \item \textbf{Recherche cosinus} dans ChromaDB :
        \begin{equation}
        \text{sim}(\mathbf{e}_q, \mathbf{e}_{d_i})
        = \frac{\mathbf{e}_q \cdot \mathbf{e}_{d_i}}
               {\|\mathbf{e}_q\| \cdot \|\mathbf{e}_{d_i}\|}
        \label{eq:cosine}
        \end{equation}
        Les $k=5$ documents $\{d_1, \ldots, d_k\}$ ayant le score cosinus le plus
        élevé sont retenus comme contexte.

  \item \textbf{Génération} : le LLM Labess-7B produit la réponse :
        \begin{equation}
        \hat{y} = \text{LLM}\bigl(\text{prompt}(q, \{d_1, \ldots, d_k\})\bigr)
        \label{eq:rag_gen}
        \end{equation}
\end{enumerate}

\section{Pipeline DVC + MLflow}
% ─────────────────────────────────────────────────────────────

\begin{lstlisting}[style=pythonstyle,
  caption={train\_anomaly.py — Pipeline MLflow + IsolationForest},
  label={lst:train_anomaly_ch5}]
import mlflow, joblib
from sklearn.ensemble import IsolationForest
from sklearn.metrics import silhouette_score

def main():
    X = pd.read_csv("data/processed/features.csv")[FEATURES].dropna().values

    with mlflow.start_run(run_name="IsolationForest_v3"):
        clf = IsolationForest(n_estimators=150, contamination=0.05,
                              random_state=42, n_jobs=-1)
        clf.fit(X)

        labels = clf.predict(X)        # -1 anomalie, +1 inlier
        sil    = silhouette_score(X, labels)

        mlflow.log_param("n_estimators",  150)
        mlflow.log_param("contamination", 0.05)
        mlflow.log_metric("silhouette_score", round(sil, 4))
        mlflow.log_metric("anomaly_rate", (labels==-1).mean())

        joblib.dump(clf, "models/anomaly_detector.pkl")
        mlflow.sklearn.log_model(clf, "model",
            registered_model_name="TelemetryAnomalyDetector")
\end{lstlisting}

\begin{tcolorbox}[warningbox, title={Limites identifiées et recommandations}]
\begin{itemize}
  \item \textbf{Modèle insects (mAP50=0.847)} : datasets publics peu représentatifs
        des espèces tunisiennes locales. \textit{Recommandation :} constituer un
        dataset photographique local avec l'AVFA.
  \item \textbf{Latence LLM (15--25 s)} : incompatible avec interaction mobile temps
        réel. \textit{Mitigation :} fallback Groq API ($<$3 s), Lite Mode Oracle.
  \item \textbf{IsolationForest statique} : pas de réentraînement incrémental.
        \textit{Recommandation :} pipeline mensuel automatique via DVC + GitHub Actions.
\end{itemize}
\end{tcolorbox}

\section{Conclusion}
% ─────────────────────────────────────────────────────────────

Ce chapitre a présenté les fondements mathématiques des trois modèles principaux
(YOLO v11, IsolationForest, RAG) et leurs résultats obtenus. Le mAP50 global de
\textbf{0.891} sur 40 classes, le Silhouette Score de \textbf{0.73} pour la détection
d'anomalies IoT, et la couverture de tests de \textbf{86\%} valident la robustesse
scientifique et technique de \farmAI{} v3.0. Le chapitre suivant présente la réalisation
concrète des interfaces et les résultats de tests intégration.
